# Option A

Смысл данной опции заключается в том, что на ASBR-рах поднимаются отдельные VRF для каждого клиентского VPN и создается сабинтерфейс, через который происходит обмен **чистыми IP маршрутами** и, через эти же сабинтерфейсы, происходит форвардинг клиентского трафика.\
Никакого MPLS между ASBR нет.\
Взаимодействие между ASBR происходит, как между PE и CE маршрутизаторами.

Недостатки:
- необходимо помимо создания vrf на PE маршрутизаторах, создавать vrf на ASBR-ах;
- для обмена маршрутами между ASBR необходимо в каждой vrf поднимать протокол маршрутизации, естественно большое количество, к примеру, bgp-сессий не благоприятно сказывается на производительности маршрутизатора;

Плюсы:
- так как между ASBR идет чистый ip трафик, без MPLS, то можно реализовать QOS и фильтрацию на основе IP заголовка; 
- трафик клиентов четко разделен; 
- данная опция является самой защищенной из всех (к примеру, если вы можете поднять данную опцию между разными провайдерами, если не хотите инжектировать чужие маршруты в свою автономную систему).

Однако, минусы в данном случае перевешивают плюсы (представьте, что вам надо прокинуть 50-60 vpn-ов – желание использовать опцию А сразу же отпадает).\

Вначале подумал, что тут вообще какая-то муть, ибо там просто все протоколы (OSPF, RIP, ISIS, EIGRP, BGP), но потом вроде разобрался и стало понятнее.

Предоставлена схема из ссылки выше и большая часть конфига:
```aiignore
RIP, EIGRP, LDP, and VPNv4 is already pre-configured. 
The inter-AS configuration between R1 and XR1 is not configured yet. 
Use option A so that CEs R9/R10, and R7/R8 have reachability. 
Use VLAN 30 for VPN_A, and VLAN 40 for VPN_B.
```

Немного подправил саму схему, добавив, где ISIS, а где OSPF в качестве underlay используется (это все видно в конфигах). И также добавил, где нужно построить VPNv4 сессии:

Ну и дальше делаем лабу :)\
По сути, на ASBRах настроить VRFы, настроить сабинтерфейсы (они уже созданы в начальном конфиге) и построить VPNv4 сессии в BGP.\
Не ошибиться в RT/RD - можно/нужно подсмотреть, что преднастроено на R2 для настройки R1 и XR2 для настройки XR1.

```aiignore
Есть еще особенность софта XR - нужно явно разрешить обмен маршрутами c помощью route-policy PASS.
```

Ну и следовательно, под каждый новый VRF нужно создавать новый сабинтерфейт, сам VRF создать и создать BGP VPNv4 сессию. Ну и маршруты каждого VRF будут инсталлированы в FIB.

Также плюс этого "винегрета" в лабе с разными протоколами в том, что обнаружили еще один минус данной опции:
```aiignore

Loss of extcommunities for OSPF and EIGRP routes. 
These routes will appear as external between CEs in different SPs, because the extcommunity attributes that have encoded the OSPF/EIGRP attributes are lost at the ASBR-ASBR link
```

Это можно поправить решением, которым вряд ли придется воспользоваться в реальной жизни, использовать EIGRP между ASBRами.

Router-LSA

![[Pasted image 20260414163749.png]]

![[Pasted image 20260414163734.png]]

В Type-1 LSA (Router LSA) используется **4 типа линков** (связей), которые описывают, как маршрутизатор подключен к другим устройствам и сетям внутри своей области.

Вот полная таблица для удобства запоминания:

| Тип   | Название            | Что означает                                                                                                                                                                                                                                                                                                                                                                                                    | Link ID (Идентификатор связи)                                                                                                                                                                                                                                                               | Link Data (Данные связи)                                                                                                                                                                      |
| ----- | ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1** | **Point-to-Point**  | Соединение "точка-точка" с другим маршрутизатором (например, через PPP, HDLC или выделенную линию)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://learningnetwork.cisco.com/s/question/0D56e0000CYKYShCQP/ospf-router-lsa-type1-lsa-deep-dive).                                                                                                                 | Router ID соседнего маршрутизатора[](https://xmind.app/mindmap/ospf-lsa-types-link-state-database/gQ5p/)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa).                                                                                              | IP-адрес локального интерфейса[](https://en.wikipedia.org/wiki/Link-state_advertisement).                                                                                                     |
| **2** | **Transit Network** | Подключение к широковещательному сегменту (например, Ethernet), где выбран Designated Router (DR)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://learningnetwork.cisco.com/s/question/0D56e0000CYKYShCQP/ospf-router-lsa-type1-lsa-deep-dive).                                                                                                                  | IP-адрес DR на этом сегменте[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://en.wikipedia.org/wiki/Link-state_advertisement).                                                                                                                | IP-адрес локального интерфейса маршрутизатора[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://en.wikipedia.org/wiki/Link-state_advertisement). |
| **3** | **Stub Network**    | Подключение к "тупиковой" сети, где нет других OSPF-соседей. Сюда попадают loopback-интерфейсы и пассивные интерфейсы[](https://blog.ipspace.net/kb/tag/OSPF/Type-1-LSA/)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa).                                                                                                                                                 | IP-номер сети (адрес сети)[](https://xmind.app/mindmap/ospf-lsa-types-link-state-database/gQ5p/)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa).                                                                                                      | Маска подсети (Network Mask)[](https://en.wikipedia.org/wiki/Link-state_advertisement)[](https://www.doit.com.cn/article/2012-03-21/5122029_5.shtml).                                         |
| **4** | **Virtual Link**    | Виртуальное соединение, используемое для подключения области к зоне 0 (backbone) через транзитную область, когда физического подключения нет[](https://xmind.app/mindmap/ospf-lsa-types-link-state-database/gQ5p/)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://learningnetwork.cisco.com/s/question/0D56e0000CYKYShCQP/ospf-router-lsa-type1-lsa-deep-dive). | Router ID соседнего маршрутизатора на другом конце виртуального линка[](https://xmind.app/mindmap/ospf-lsa-types-link-state-database/gQ5p/)[](https://networklessons.com/cisco/ccnp-enarsi-300-410/ospfv2-lsa-type-1-router-lsa)[](https://en.wikipedia.org/wiki/Link-state_advertisement). | IP-адрес локального интерфейса (иногда внутренний идентификатор)[](https://en.wikipedia.org/wiki/Link-state_advertisement).                                                                   |

## Термины

#MPU - *Main Processing Unit,* то есть главная плата управления. Отвечает за плоскость контроля и плоскость управления системой, включая расчет маршрута, управление устройствами и техническое обслуживание, а также мониторинг устройств. В качестве системного блока синхронизации он обеспечивает высокоточную и высоконадежную синхронизацию тактовых и временных сигналов. Главная панель управления подобна уровню управления почтового отделения и не отвечает за конкретную работу по экспресс-обработке (пересылке пакетов). MPU интерпретирует конфигурацию устройства и затем передает ее в LPU.

#LPU - *Line Processing Unit*, который также называют интерфейсной платой. Интерфейсная плата со многими интерфейсами - это интерфейсная плата. Пакеты данных принимаются и отправляются через интерфейсную плату. Кабели связи должны быть подключены к интерфейсам интерфейсной платы. Интерфейсная плата обеспечивает различные типы интерфейсов (например, оптические и электрические интерфейсы) и различные скорости (например, 1000 Мбит/с, 10 Гбит/с, 25 Гбит/с, 40 Гбит/с и 100 Гбит/с).

#SFU - *Switch Fabric Unit*. SFU обеспечивает [[data plane|data plane]] для всей системы. LPU и MPU взаимодействуют друг с другом через SFU.

## Процесс работы
```
MPU управляет LPU и SFU через системную шину (например, HiGig/HiGig2 в Huawei).

LPU принимает/передаёт трафик через физические порты (SFP+, QSFP28).

SFU соединяет все LPU между собой, обеспечивая коммутацию на высокой скорости (Tbps)
```

## Для чего нужна шина между MPU и SFU?

1. Управление коммутационной матрицей (конфигурация и мониторинг SFU):
- Настройки путей передачи данных (например, приоритезация QoS).
- Таблицы агрегации и балансировки нагрузки (если SFU поддерживает динамическое распределение трафика).
- Политики аварийного переключения (например, при отказе линии).

2. Синхронизация FIB и RIB при изменениях маршрутизации:
- MPU хранит RIB и вычисляет FIB, но для быстрой пересылки FIB должна быть актуальной на всех LPU.
- SFU помогает в синхронизации:
	- При изменении маршрута (например, BGP update) MPU обновляет FIB и через SFU рассылает изменения на LPU.
	- В некоторых архитектурах SFU участвует в широковещательном обновлении FIB на все линейные карты.

3. Обеспечение отказоустойчивости (redundancy)
- В устройствах с двумя MPU (активный/резервный) SFU обеспечивает:
	- Бесперебойную коммутацию при переключении между MPU.
	- Синхронизацию состояний (например, сессий BGP) через SFU.
```
Пример: в Huawei NE5000E при падении основного MPU резервный мгновенно берет управление SFU без потери пакетов.
```

## А зачем тогда шина между MPU и LPU?

*Шина MPU ↔️ LPU – для управления (control plane).*

*SFU – для передачи трафика и FIB (data plane).*

_Оба канала не заменяют, а дополняют друг друга._

1. Управление и контроль LPU (шина MPU ↔️ LPU)

Эта шина (Management Bus, например, I2C, SPI, PCIe) нужна для:
- Конфигурации интерфейсов (включение/выключение портов, настройка VLAN, ACL).
- Мониторинга состояния (температура, ошибки, нагрузка на порты).
- Загрузки ПО/Firmware на LPU (например, при обновлении).
- Аварийного управления (например, перезагрузка LPU без участия SFU).

```
Пример: Если SFU вышел из строя, MPU всё равно может управлять LPU через эту шину.
```

2. Передача трафика и синхронизация FIB (через SFU) SFU используется для:

- Высокоскоростной пересылки пакетов между LPU (data plane).
- Распространения обновлений FIB (но не конфигурации!).

```
Пример: Когда MPU вычисляет новый маршрут, он загружает FIB в LPU через SFU, чтобы не нагружать управляющую шину.
```
==Eсли прилетел пакет, например, ARP.
Он прилетит на LPU - дальше этот пакет будет передан через SFU или напрямую на MPU (если напрямую, то через какую шину)?:==

- **Обычный трафик (data-plane) → SFU.
- **Управляющие пакеты (ARP, BFD, OAM) → Control Bus (MPU ↔️ LPU).
- **Гибридные случаи (например, Proxy ARP) могут использовать оба пути.

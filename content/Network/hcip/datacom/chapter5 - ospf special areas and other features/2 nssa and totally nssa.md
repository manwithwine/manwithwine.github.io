
**`Problems in the Stub Area and Totally Stub Area:`**
As defined in OSPF, the stub area cannot import external routes. This mechanism prevents a large number of external routes from consuming resources of devices in the stub area.

Stub and totally stub areas **cannot apply** to the scenario where external routes need to be imported and resource consumption caused by external routes also needs to be avoided.

## NSSA and Totally NSSA
An NSSA **can import** external routes but **cannot learn external** routes imported from other areas on the OSPF network.
The difference between the totally NSSA and NSSA is that no-summary is specified on the ABR for the totally NSSA.

![[Pasted image 20260415114220.png]]

Type 7 LSAs are generated by an Autonomous System Boundary Router (ASBR) in an NSSA and advertised only within the NSSA. After an ASBR in an NSSA imports external routes to the NSSA, the ASBR generate Type 7 LSAs to carry information about the external routes.

The Type 7 LSAs are advertised **only within the NSSA**.

When Type 7 LSAs reach an ABR in the NSSA, the ABR **translates the Type 7 LSAs into Type 5 LSAs**, imports them to the backbone area, and floods them to the entire AS.
he ABR in an NSSA prevents external routes imported from other areas from being imported into the NSSA. That is, Type 4 and Type 5 LSAs do not exist in the NSSA. To enable routers in the NSSA to reach the AS external network through the backbone area, the ABR in the NSSA **automatically injects a default route carried by a Type 7 LAS** into the NSSA.

**`The difference is that the ABR in a totally NSSA prevents Type 3 LSAs from entering the totally NSSA.`**

# Типы LSA в OSPF

## Что такое LSA?

**LSA (Link State Advertisement)** - это фундаментальная единица информации в OSPF, которая описывает часть сетевой топологии. Каждый LSA содержит информацию о каналах маршрутизатора, сетях, стоимостях и других параметрах.

## Структура LSA

Все типы LSA имеют общий 20-байтный заголовок:
**Ключевые поля:**
- **LS Age** - возраст LSA в секундах
- **LS Type** - тип LSA (1-7, 9-11)
- **Link State ID** - идентификатор, зависящий от типа LSA
- **Advertising Router** - Router ID источника LSA
- **LS Sequence Number** - номер последовательности для определения актуальности
- **LS Checksum** - контрольная сумма содержимого LSA

## Основные типы LSA

| Тип   | Название             | Описание                                                |
|:------|:---------------------|:--------------------------------------------------------|
| 1     | **Router LSA**       | Описывает связи маршрутизатора внутри области           |
| 2     | **Network LSA**      | Создается DR, описывает всех маршрутизаторов в сегменте |
| 3     | **Summary LSA**      | Генерируется ABR, передает информацию между областями   |
| 4     | **ASBR Summary LSA** | Указывает путь к ASBR                                   |
| 5     | **External LSA**     | Распространяет внешние маршруты                         |
| 7     | **NSSA LSA**         | Для NSSA областей (Not-So-Stubby Area)                  |

### Тип 1: Router LSA

**Назначение:** Описывает каналы маршрутизатора в пределах одной области.

**Генерируется:** Каждым маршрутизатором для каждой области, к которой он подключен

**Распространение:** Только в пределах своей области

**Содержит:**
- Типы подключенных каналов (point-to-point, transit, stub, virtual)
- Стоимости каналов
- IP-адреса подключенных сетей
- Флаги (ABR, ASBR, Virtual Link endpoint)

**Особенности:** Непрозрачны между областями - ABR преобразует их в Network/Summary LSA

### Тип 2: Network LSA

**Назначение:** Описывает multi-access сети и подключенные к ним маршрутизаторы.

**Генерируется:** Только DR (Designated Router) в multi-access сети

**Распространение:** Только в пределах своей области

**Содержит:**
- IP-маску сети
- Список всех маршрутизаторов, подключенных к сети (включая DR)
- Стоимость подключения к сети

**Особенности:** Существует только для broadcast и NBMA сетей

### Тип 3: Summary LSA

**Назначение:** Распространяет маршруты между областями (inter-area routes).

**Генерируется:** ABR (Area Border Router)

**Распространение:** Из одной области в другие через ABR

**Содержит:**
- IP-адрес сети (суммированной)
- Маску сети
- Стоимость до сети из ABR

**Типы Summary LSA:**
- **Network Summary** - для IPv4 сетей
- **ASBR Summary** - для маршрутов к ASBR

### Тип 4: ASBR Summary LSA

**Назначение:** Сообщает о расположении ASBR (Autonomous System Boundary Router).

**Генерируется:** ABR

**Распространение:** Во все области, кроме stub-областей

**Содержит:**
- Router ID ASBR
- Стоимость до ASBR из ABR

**Важность:** Позволяет маршрутизаторам в других областях узнавать путь к ASBR для внешних маршрутов

### Тип 5: AS External LSA

**Назначение:** Распространяет внешние маршруты (из других протоколов) по всей AS OSPF.

**Генерируется:** ASBR

**Распространение:** Во все области, кроме stub-областей

**Содержит:**
- Внешний IP-адрес сети
- Маску сети
- Стоимость (E1 или E2)
- Адрес внешнего next hop

**Типы метрик:**
- **E1** - стоимость включает внутреннюю стоимость до ASBR + внешнюю стоимость
- **E2** - используется только внешняя стоимость (по умолчанию)

### Тип 7: NSSA External LSA

**Назначение:** Распространяет внешние маршруты в NSSA (Not-So-Stubby Area).

**Генерируется:** ASBR в NSSA области

**Распространение:** Только в пределах NSSA области

**Преобразование:** ABR преобразует Type 7 в Type 5 при выходе из NSSA

**Особенности:** Позволяет импортировать внешние маршруты в stub-области

## Дополнительные типы LSA (Opaque LSA)

### Тип 9: Link-Local Opaque LSA
**Распространение:** Только в пределах одного канала
**Использование:** Для MPLS TE, Graceful Restart

### Тип 10: Area-Local Opaque LSA
**Распространение:** В пределах одной области
**Использование:** Для дополнительных сервисов в области

### Тип 11: AS-Wide Opaque LSA
**Распространение:** По всей AS OSPF
**Использование:** Для доменных сервисов

## Процесс старения и обновления LSA

### LS Refresh
- Каждые **30 минут** оригинатор пересылает LSA
- Обновляет sequence number
- Подтверждает актуальность информации

### Max Age
- LSA удаляется из LSDB после **60 минут**
- Если оригинатор недоступен, LSA постепенно исчезает из сети

### Flooding Process
- При изменении LSA распространяется через flooding
- Каждый маршрутизатор проверяет актуальность и пересылает соседям

## Иерархия предпочтения LSA

При получении нескольких LSA для одной сети предпочтение отдается:
1. **Router LSA (Type 1)** - самая точная информация
2. **Network LSA (Type 2)** - информация от DR
3. **Summary LSA (Type 3)** - межобластные маршруты
4. **External LSA (Type 5/7)** - внешние маршруты

## Влияние на проектирование сети

**Правильное понимание LSA позволяет:**
- Оптимизировать размер LSDB
- Снизить нагрузку на маршрутизаторы
- Правильно проектировать иерархию областей
- Эффективно фильтровать маршруты

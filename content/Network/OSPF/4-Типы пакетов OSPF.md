# Типы пакетов OSPF

## Общая информация о пакетах OSPF

Все пакеты OSPF используют IP-протокол номер 89 и имеют общий заголовок. OSPF использует пять типов пакетов, каждый из которых выполняет специфическую роль в процессе обмена информацией и поддержания работоспособности протокола.

## Структура общего заголовка OSPF

Каждый пакет OSPF начинается с 24-байтного заголовка.


**Поля заголовка:**
- **Version #** - версия OSPF (2 для IPv4)
- **Type** - тип пакета (1-5)
- **Packet Length** - общая длина пакета
- **Router ID** - идентификатор отправителя
- **Area ID** - идентификатор области
- **Checksum** - контрольная сумма пакета
- **AuType** - тип аутентификации
- **Authentication** - данные аутентификации

## Тип 1: Hello-пакеты

### Назначение
Обнаружение соседей, поддержание смежностей и выбор DR/BDR.

### Ключевые поля
- **Hello Interval** - как часто отправляются Hello-пакеты
- **Dead Interval** - время ожидания перед объявлением соседа недоступным
- **Network Mask** - маска сети интерфейса
- **Neighbor List** - список видимых соседей
- **Router Priority** - приоритет для выбора DR/BDR
- **DR/BDR IP Address** - адреса текущих DR и BDR

### Особенности работы
- Отправляются **мультикастом** на 224.0.0.5 (AllSPFRouters)
- В NBMA-сетях отправляются **юникастом**
- Получение Hello-пакета подтверждает работоспособность соседа

## Тип 2: Database Description (DBD) пакеты

### Назначение
Синхронизация баз данных LSDB между маршрутизаторами.

### Ключевые поля
- **Sequence Number** - порядковый номер для синхронизации
- **Flags** (I, M, MS):
  - **I** (Init) - первый пакет в обмене
  - **M** (More) - ожидаются еще пакеты
  - **MS** (Master/Slave) - роль в обмене
- **LSA Headers** - заголовки LSA из базы данных

### Процесс обмена DBD
1. **Выбор Master/Slave** - маршрутизатор с более высоким RID становится Master
2. **Последовательный обмен** - Master управляет порядком обмена
3. **Сравнение LSDB** - выявление различий в базах данных

## Тип 3: Link State Request (LSR) пакеты

### Назначение
Запрос конкретных LSA после обнаружения различий в DBD-обмене.

### Ключевые поля
- **LS Type** - тип запрашиваемого LSA
- **Link State ID** - идентификатор LSA
- **Advertising Router** - маршрутизатор-источник LSA

### Особенности работы
- Отправляются только при обнаружении устаревших или отсутствующих LSA
- Каждый LSR запрашивает один конкретный LSA
- Процесс продолжается до полной синхронизации LSDB

## Тип 4: Link State Update (LSU) пакеты

### Назначение
Доставка запрошенных или обновленных LSA.

### Ключевые поля
- **Number of LSAs** - количество LSA в пакете
- **LSA Instances** - сами объявления о состоянии каналов

### Особенности работы
- Могут содержать несколько LSA в одном пакете
- Отправляются как ответ на LSR или при изменениях топологии
- Используют надежную доставку через подтверждения

## Тип 5: Link State Acknowledgment (LSAck) пакеты

### Назначение
Подтверждение получения LSU-пакетов.

### Ключевые поля
- **LSA Headers** - заголовки подтверждаемых LSA

### Особенности работы
- Могут подтверждать несколько LSA одновременно
- Отправляются мультикастом в broadcast-сетях
- Гарантируют надежную доставку LSA

## Процесс взаимодействия пакетов

### Пример полного обмена:
1. **Hello** → Обнаружение соседей
2. **DBD** → Сравнение оглавлений LSDB
3. **LSR** → Запрос недостающих LSA
4. **LSU** → Отправка запрошенных LSA
5. **LSAck** → Подтверждение получения

### Тайминги и повторные передачи
- **Retransmission Interval** - таймаут перед повторной отправкой LSU
- **Каждый неподтвержденный LSU** хранится в retransmission list
- **Повторная отправка** происходит до получения LSAck или разрыва смежности

## Особенности в разных типах сетей

### Broadcast-сети
- **Hello, LSU, LSAck** - мультикаст на 224.0.0.5/224.0.0.6
- **DBD, LSR** - юникаст

### Point-to-Point сети
- Все пакеты отправляются мультикастом на 224.0.0.5

### NBMA-сети
- Все пакеты отправляются юникастом на явно указанных соседей

# Virtual Link (Виртуальное соединение в OSPF)

## Зачем нужен Virtual Link
Все области OSPF должны быть связаны с **Backbone (Area 0)**.  
Если одна область не имеет прямого соединения с Backbone, можно создать **Virtual Link** через другую область.

## Принцип работы
- Virtual Link — это **логический туннель** между двумя ABR.
- Используется **IP внутри транзитной области**, но принадлежит **Area 0**.
- Соседство устанавливается как обычное OSPF-соединение.

## Пример
```
Area 0 ←→ Area 1 ←→ Area 2
```
Если Area 2 не имеет прямого выхода в Area 0, то через ABR в Area 1 создается Virtual Link:
```
area 1 virtual-link <Router-ID>
```

# База данных состояния каналов (LSDB)

## Что такое LSDB?

**LSDB (Link State Database)** - это полная топологическая карта сети OSPF, на основе которой алгоритм SPF вычисляет кратчайшие пути ко всем узлам сети.

## Структура LSDB

### Организация данных
LSDB организована по областям (areas). Каждая область имеет свою собственную LSDB.

### Ключевые характеристики LSDB

- **Синхронизированность**: Все маршрутизаторы в одной области имеют идентичную LSDB
- **Иерархичность**: Разные области имеют разные LSDB
- **Масштабируемость**: Разделение на области ограничивает размер LSDB

## Компоненты LSDB

### Для внутренних маршрутизаторов (в одной области)
- **Router LSA (Type 1)** всех маршрутизаторов области
- **Network LSA (Type 2)** всех multi-access сетей области

### Для ABR (Area Border Router)
- **Полная LSDB** для каждой подключенной области
- **Summary LSA (Type 3)** для маршрутизации между областями
- **ASBR Summary LSA (Type 4)** для указания местоположения ASBR

### Для ASBR (AS Boundary Router)
- Все вышеперечисленное плюс
- **AS External LSA (Type 5)** для распространения внешних маршрутов

## Процесс синхронизации LSDB

### Initial Synchronization
При установлении смежности маршрутизаторы синхронизируют свои LSDB:

1. **Обмен DBD пакетами** - сравнение оглавлений LSA
2. **Отправка LSR** - запрос недостающих или устаревших LSA
3. **Отправка LSU** - доставка запрошенных LSA
4. **Подтверждение LSAck** - завершение синхронизации

### Flooding Process
При изменении топологии происходит распространение обновлений:
```aiignore
Маршрутизатор A обнаруживает изменение
↓
Генерирует обновленное LSA
↓
Отправляет LSU соседям (кроме источника)
↓
Каждый сосед проверяет актуальность
↓
Если LSA новее - обновляет LSDB и пересылает дальше
↓
Процесс продолжается до полного распространения
```

## Типы записей в LSDB

### Self-Originated LSA
LSA, созданные самим маршрутизатором. За эти записи маршрутизатор отвечает:
- Периодическое обновление (каждые 30 минут)
- Удаление при прекращении существования
- Обновление при изменениях

### Received LSA
LSA, полученные от других маршрутизаторов:
- Хранятся в LSDB
- Пересылаются соседям при изменениях
- Удаляются при истечении MaxAge

### Stub Network Information
Информация о заглушенных сетях:
- Сети без других маршрутизаторов
- Описываются в Router LSA
- Не требуют отдельных LSA

## Мониторинг и анализ LSDB

### Ключевые параметры для мониторинга

**Размер LSDB:**
- Количество LSA каждого типа
- Общий объем памяти, занимаемый LSDB
- Скорость роста базы данных

**Стабильность LSDB:**
- Частота изменений LSA
- Количество flooded обновлений
- Стабильность sequence numbers

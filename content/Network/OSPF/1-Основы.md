

# OSPF: Основы

## Что такое OSPF?

**OSPF (Open Shortest Path First)** - это протокол динамической маршрутизации, который относится к классу **Link-State Protocol**. В отличие от дистанционно-векторных протоколов (как RIP), OSPF строит полную топологическую карту сети и вычисляет оптимальные пути с помощью алгоритма Дейкстры.

## Ключевые характеристики

- **Открытый стандарт**: Описан в RFC 2328
- **Классовый протокол (Classless)**: Поддерживает VLSM и CIDR
- **Метрика**: Стоимость (Cost), вычисляемая на основе пропускной способности
- **Административное расстояние**: 110
- **Алгоритм**: SPF (Shortest Path First) или алгоритм Дейкстры
- **Транспортный протокол**: IP (протокол 89)
- **Мультикаст-адреса**: 224.0.0.5 (ALLSPFRouters), 224.0.0.6 (ALLDRRouters)

## Основные термины OSPF

### Router ID (RID)
**Уникальный идентификатор маршрутизатора** в OSPF-домене. Определяется в порядке приоритета:
1. Явно заданный ID через команду `router-id`
2. Наивысший IP-адрес loopback-интерфейса
3. Наивысший IP-адрес активного физического интерфейса

### Cost (Стоимость)
**Метрика OSPF**, вычисляемая на основе пропускной способности канала:
```aiignore
Cost = Reference Bandwidth / Interface Bandwidth
По умолчанию Reference Bandwidth = 100 Мбит/с
```

### LSA (Link State Advertisement)
**Объявление о состоянии канала** - фундаментальная единица информации в OSPF. Каждый маршрутизатор рассылает LSA, описывающие его интерфейсы, соседей и сети.

### LSU (Link State Update)
**Обновление состояния канала** - пакет, который переносит один или несколько LSA.

### LSR (Link State Request)
**Запрос состояния канала** - запрос на отправку конкретного LSA.

### LSDB (Link State Database)
**База данных состояния каналов** - коллекция всех LSA, полученных маршрутизатором. Все маршрутизаторы в одной области имеют идентичную LSDB.

## Принципы работы OSPF

1. **Установление соседства** - маршрутизаторы обнаруживают друг друга
2. **Обмен LSA** - маршрутизаторы описывают свою часть сети
3. **Построение LSDB** - формируется полная карта сети
4. **Выполнение алгоритма SPF** - вычисление кратчайших путей
5. **Заполнение таблицы маршрутизации** - добавление лучших маршрутов

## Преимущества OSPF

- **Быстрая сходимость** - изменения в топологии быстро распространяются
- **Масштабируемость** - поддержка иерархической структуры через области
- **Отсутствие петель** - благодаря алгоритму SPF
- **Эффективное использование полосы** - обновления рассылаются только при изменениях
- **Поддержка VLSM/CIDR** - полная поддержка бесклассовой адресации

## Недостатки OSPF

- **Высокие требования к ресурсам** - процессор и память для алгоритма SPF
- **Сложность конфигурации** - по сравнению с дистанционно-векторными протоколами
- **Требовательность к проектированию** - особенно в крупных сетях

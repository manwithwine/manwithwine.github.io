# Области (Areas) и их типы

OSPF поддерживает иерархическую структуру областей.

## Основная идея
- Разделение сети на области снижает нагрузку на процессор и объем LSDB.
- Вся маршрутизация между областями проходит через **Backbone Area (0)**.

## Типы областей

| Тип                           | Поддерживаемые LSA   | Особенности                                                                 |
|:------------------------------|:---------------------|:----------------------------------------------------------------------------|
| **Normal Area**               | Все типы (1–5)       | Полная таблица маршрутов                                                    |
| **Stub Area**                 | Только 1, 2, 3       | Не получает внешние маршруты (LSA Type 5)                                   |
| **Totally Stubby Area**       | Только 1, 2          | Вместо межобластных маршрутов — только Default Route                        |
| **NSSA (Not-So-Stubby Area)** | 1, 2, 3, 7           | Позволяет импортировать внешние маршруты как Type 7, но не принимает Type 5 |

## Работа с разными LSA типами (3, 5, 7)

### Type 3 — Summary LSA
- Создается **ABR**.
- Передает информацию о маршрутах из одной области в другую.

### Type 5 — External LSA
- Создается **ASBR** для маршрутов, импортированных из других протоколов (BGP, RIP и т.д.).
- Распространяется по всей OSPF сети, кроме Stub областей.

### Type 7 — NSSA External LSA
- Используется внутри **NSSA области**.
- Генерируется ASBR в NSSA.
- При передаче в Backbone, ABR **переводит Type 7 → Type 5**.

#### Пример
```
ASBR в Area 10 (NSSA) импортирует маршрут 192.168.100.0/24.
→ Создает LSA Type 7.
→ ABR конвертирует его в LSA Type 5 для распространения в другие области.
```

# Алгоритм SPF (Shortest Path First)

## Что такое алгоритм SPF?

**Алгоритм SPF (Shortest Path First)**, также известный как **алгоритм Дейкстры**, - это математический алгоритм, который OSPF использует для вычисления кратчайших путей от исходного маршрутизатора ко всем остальным узлам в сети на основе информации из LSDB.

## Основные понятия алгоритма

### Граф сети
- **Вершины (Nodes)**: Маршрутизаторы и сети
- **Рёбра (Edges)**: Каналы связи между маршрутизаторами
- **Вес (Cost)**: Метрика, основанная на пропускной способности канала

### Ключевые множества
- **TENTATIVE**: Кандидаты на рассмотрение (временный список)
- **PATHS**: Подтверждённые кратчайшие пути (постоянный список)

## Пошаговый процесс выполнения алгоритма SPF

### Шаг 1: Инициализация
- Поместить локальный маршрутизатор в множество **PATHS** с cost = 0
- Добавить всех непосредственных соседей в множество **TENTATIVE** с соответствующими cost

### Шаг 2: Выбор следующей вершины
- Найти в **TENTATIVE** вершину с наименьшим cost
- Переместить эту вершину из **TENTATIVE** в **PATHS**
- Эта вершина становится новой "текущей" для расширения

### Шаг 3: Расширение графа
- Для каждой вершины, подключенной к текущей:
  - Вычислить общий cost: `cost_текущий + cost_канала`
  - Если вершины нет в **TENTATIVE** - добавить
  - Если вершина уже в **TENTATIVE** - обновить cost, если новый путь короче

### Шаг 4: Повторение
- Повторять шаги 2-3 до тех пор, пока множество **TENTATIVE** не станет пустым

## Пример выполнения алгоритма

### Исходные данные:
```aiignore
Маршрутизаторы: A, B, C, D
Каналы:
A-B: cost 5
A-C: cost 10
B-C: cost 2
B-D: cost 1
C-D: cost 3
```

### Процесс расчета от маршрутизатора A:

**Итерация 1:**
- PATHS: A(0)
- TENTATIVE: B(5), C(10)
- Выбираем B(5) - наименьший cost

**Итерация 2:**
- PATHS: A(0), B(5)
- TENTATIVE: C(7), D(6)
- Выбираем D(6) - наименьший cost

**Итерация 3:**
- PATHS: A(0), B(5), D(6)
- TENTATIVE: C(7)
- Выбираем C(7) - последняя вершина

**Результат:** Кратчайшие пути от A:
- A→B: cost 5
- A→D: cost 6 (через B)
- A→C: cost 7 (через B)

## Типы SPF-расчетов в OSPF

### Full SPF Calculation
Полный перерасчет дерева кратчайших путей. Запускается при:
- Изменении Router LSA (Type 1)
- Изменении Network LSA (Type 2)
- Добавлении/удалении маршрутизатора в области

### Partial SPF Calculation
Частичный перерасчет. Запускается при:
- Изменении Summary LSA (Type 3)
- Изменении межобластных маршрутов

### External Route Calculation
Расчет внешних маршрутов. Запускается при:
- Изменении AS External LSA (Type 5)
- Изменении NSSA External LSA (Type 7)

## SPF-таймеры и оптимизация

### SPF Hold Time
Минимальное время между последовательными запусками SPF:
```aiignore
SPF Hold Time = spf-start + (spf-hold × n)
```

Где:
- **spf-start**: Начальная задержка (обычно 0 сек)
- **spf-hold**: Базовая задержка (обычно 5 сек)  
- **n**: Количество последовательных расчетов

### Экспоненциальное торможение
При частых изменениях таймер увеличивается для стабилизации сети:
- 1-й расчет: 0 сек
- 2-й расчет: 5 сек
- 3-й расчет: 10 сек
- 4-й расчет: 20 сек
- Максимум: 10 сек (в некоторых реализациях)

## Построение дерева кратчайших путей

### Результат выполнения SPF
Алгоритм строит **SPF-дерево** с корнем в локальном маршрутизаторе:
```
    A (0)
   / \
 5/   \10
 /     \
B(5)   C(7)
 \     /
 1\   /3
   \ /
   D(6)
```

### Заполнение таблицы маршрутизации
На основе SPF-дерева заполняется таблица маршрутизации:
- **Next-hop**: Следующий маршрутизатор на пути к цели
- **Interface**: Исходящий интерфейс
- **Cost**: Суммарная стоимость пути

## Особенности расчета для разных типов LSA

### Router LSA (Type 1)
- Представляют маршрутизаторы как вершины графа
- Содержат информацию о подключенных каналах
- Используются для построения основного графа

### Network LSA (Type 2)  
- Представляют multi-access сети как псевдо-вершины
- Позволяют корректно рассчитать пути через broadcast-сети
- Учитывают роль DR в расчетах

### Summary LSA (Type 3)
- Не участвуют в построении графа
- Добавляются как листья SPF-дерева
- Учитываются после построения основного графа
